## REST api
- 웹주소를 api로 사용하는 기술
![alt text](image.png)


### 문제점
- 요청(request)-응답(response) 방식
- - 프로그래머가 특정 api 요청 -> 서버는 결과값 응답
- - 요청 횟수에 제한(ex. 10회/초)
- - 서버에서 바로 알 수 없음, 즉 지연이 발생


## 웹소켓 방식
- 서버-클라이언트 실시간 양방향 통신 기술
- - 실시간 전송을 위해 암호화폐 거래소에서 사용함
- - 요청-응답이 아닌, 구독(subscribe) 방식
- - 서버에서 체결 즉시 구독한 클라이언트(PC)에 해당 데이터 전송
